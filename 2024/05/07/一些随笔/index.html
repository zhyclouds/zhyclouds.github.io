一些随笔

每一个艺术家都是从学徒做起的

  • 通读官方文档
  • 官方blog（一些大佬的思考和最新的东西）
  • 不要去网上搜一些特别智障的问题，官方文档里面一般其实都有解决方案的
  • 读一些源码实现、底层实现
  • 总结、形成自己的知识体系

有效的思考

无论是CRUD还是做架构，其实都是一个演进的过程，在这个过程中需要不断的去思考、去看一些方法论为什么是这样的…

比如

  • 错误码如何设计？
  • 为什么是三层架构、DDD？
  • 如何更好的对代码进行抽象？
  • 有没有一些工具或者方法能够提高我的效率？
  • …

实践、踩坑、重视、反思、改进

先看别人怎么做（思考从国内到国外所有公司的架构设计），看别人的演进过程，再结合自己的实际体量，进行设计（恰到好处的设计）

这样就可以知道什么能做、什么不能做、未来出现一些情况问题如何去解决

做一个合理的设计

机会和选择

大厂螺丝钉还是小厂中坚力量？

公司是不是创业型的？企业能否高速增长？如何和企业一起成长？

一万小时定律

不断积累、不断总结

团队管理 + 向上管理

带一个小团队

讲话讲不清楚，如何向上汇报业务？

  • 不会看数据
  • 不会做总结
  • 用一两句话把重点讲出来

自我认知和修行

对自己的性格有一个认识，认识到哪些地方是好的，哪些是不好的，尽可能不要感情用事

持续学习，投资自己永远是值得的

乐于分享、多和人交流，但同时也要内敛，不要过于张扬

核心G0技能

  • Go 最关键的几个特性：goroutine、channel、error、context
  • Runtime 中核心类型：chan、slice、map
  • Go toolchain
  • 内存模型
  • 并发语义
  • 网络开发
  • 大规模项目的工程化实践
  • Kernel 综合知识对工程师代码的反向要求

构建Go语言知识体系

  • 系统性的阅读官方文档
  • 系统性的阅读官方类库
  • 阅读大量优秀开源项目
  • 持续实践并改进（应该自己去阅读代码，而不是听别人讲代码）
  • 行业内的优秀的文章（大佬）
  • 知识的组织关联

时间管理

  • 碎片化时间（看看文章）
  • 完整时间（系统的学习）
  • 工作时间
  • 业余时间

信息获取

  • 微信群
  • 公众号 + 收藏 + Tag
  • 买几本好书
  • Paper（VLDB、sigmod等）
  • MIT之类的视频
  • Meetup + 大会
  • 啤酒烧烤（高级点叫喝茶）

工具

  • OneNote
  • MarginNote3
  • Omni Graffle
